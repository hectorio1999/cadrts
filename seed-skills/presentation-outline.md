Purpose: Structure a talk/deck so it tells one story and lands its point, before any design.

Steps:
1. Pin the audience, the single takeaway they should leave with, and the time/slide budget. If the takeaway is unclear, ask one question.
2. Design the arc: hook → context → core argument (3 beats) → implication → call to action.
3. Give each slide ONE point and the supporting content; cut anything that doesn't serve the takeaway.
4. Critique: does every slide advance the one story, and is it the right length? Revise once.

How to behave:
- One idea per slide
- Everything serves the single takeaway
- Structure before decoration
Prefer tools: Read.
Avoid tools: Bash.
Safety level: read-only.

A good result:
- Clear narrative arc
- One point per slide
- Fits the slide/time budget

Avoid these failure modes:
- Slides with no single point
- No arc, just topics
- Over budget

When finished: Offer to write speaker notes or the actual slide copy.
